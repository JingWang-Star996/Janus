# 数据报表管理

## 周期性任务

### 每周五

**任务**：督促更新项目管理平台进度

**操作**：
1. 检查各组进度更新情况
2. 提醒未更新的负责人
3. 汇总本周完成情况
4. 规划下周工作安排

---

### 每月 25-27 日

**任务**：收集下月功能开发耗时占比统计

**操作**：
1. 发送数据收集通知
2. 提供统计模板
3. 收集各组数据
4. 汇总整理
5. 提交报表

**统计内容**：
- 功能开发耗时
- 可重复体验功能占比
- 下月计划

---

### 每月底

**任务**：收集项目总开发进度百分比

**操作**：
1. 收集各组进度数据
2. 计算总体进度
3. 整理月度报告
4. 提交项目负责人

---

## 数据收集类型

### 问卷数据

- 测试问卷
- 用户反馈
- 内部调研

### 埋点数据

- 用户行为数据
- 功能使用率
- 留存数据

### 市调数据

- 竞品分析
- 市场反馈
- 用户画像

### 测试反馈

- 功能测试报告
- 机型测试结果
- 云测数据

## 常用报表模板

- 其他：XX%

- [风险 2]

## 注意事项

- ⚠️ 提前 3 天发送数据收集通知
- ⚠️ 设置明确的截止时间
- ⚠️ 及时催办未提交的组
- ⚠️ 数据核对后再提交
