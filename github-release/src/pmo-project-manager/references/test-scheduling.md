# 测试排期协调流程

## 提前量参考

| 测试类型 | 提前量 | 备注 |
|----------|--------|------|
| 功能测试 | 1-2 周 | 需预约档期 |
| 机型测试 | 1-2 周 | 协调空档 |
| 云测 | 1 周 | 考虑节假日人力 |
| 市调测试 | 2-3 周 | 准备材料较多 |

---

## 发包测试准备流程

### 步骤 1：确认版本时间和测试目标

**需要确认**：
- 版本发布时间节点
- 测试目标（功能验证/性能测试/兼容性）
- 测试重点功能列表
- 预估测试时长

### 步骤 2：创建需求表并同步各方

**需求表内容**：
- 项目名称
- 测试类型
- 测试时间
- 测试设备/机型要求
- 测试重点说明
- 联系人信息

**同步对象**：
- 运营部门
- 测试中心
- 项目组成员

### 步骤 3：协调测试排期

**联系测试中心**：
- 说明测试需求和时间
- 了解可用档期
- 协调冲突（如有）
- 确认最终排期

**常见冲突处理**：
```
如果排期冲突：
1. 了解冲突原因
2. 协商调整方案
3. 寻找替代资源
4. 优化时间安排
```

### 步骤 4：跟进准备工作

**准备清单**：
- [ ] 翻译完成
- [ ] 埋点配置
- [ ] 测试素材
- [ ] 测试包
- [ ] 测试用例

### 步骤 5：协助提审和开广

**提审材料**：
- 游戏版号信息
- 隐私政策
- 用户协议
- 测试报告

**开广设置**：
- 广告平台账号
- 投放计划
- 素材审核

### 步骤 6：测试后收集数据

**收集内容**：
- 测试数据报表
- 问卷反馈
- 问题清单
- 优化建议

---

## 注意事项

### 时间规划

- ⚠️ 功能测试需提前 1-2 周预约
- ⚠️ 节假日前云测人力紧张
- ⚠️ 年底测试资源尤其紧张

### 材料准备

- ✅ 测试包提前准备好
- ✅ 测试用例写清楚
- ✅ 预期结果明确

### 沟通协调

- ✅ 提前同步所有相关方
- ✅ 变更及时通知
- ✅ 问题快速响应
