## 🎯 一句话需求

> "做一个收集表，让团队成员填写 AI 使用情况，最后生成统一格式的文本发给我整合。"

## 🚀 OpenClaw 核心能力展示

### 1. 文件生成与发送

**命令**：
```
你做完表格直接发我文件。
```

**AI 操作**：
```python
# 自动写入文件
write(path="/workspace/outputs/收集表.html")

# 自动发送给用户
message(action="send", filePath="收集表.html")
```

**效果**：聊天窗口直接收到文件，无需手动下载

---

### 2. 快速迭代开发

**6 次迭代记录**：

| 版本 | 用户反馈 | AI 响应 |
|------|----------|--------|
| V1 CSV | "格式太简陋" | 优化为 HTML 表格 |
| V2 表格 | "不够智能" | 增加动态表单 |
| V3 表单 | "加工具多选" | 12 个工具选项 |
| V4 工具 | "加时间戳" | 实时显示时间 |
| V5 时间 | "有 BUG，程序策划无法生成" | 5 分钟修复 |
| V6 修复 | ✅ "可以了" | 交付 |

**OpenClaw 优势**：
- ✅ 上下文完整保留
- ✅ 每次迭代基于历史
- ✅ 无需重复说明需求

---

### 3. BUG 定位与修复

**用户反馈**：
> "只有美术可以点击生成汇报文本，程序和策划还是不行"

**AI 诊断过程**：
```javascript
// 1. 检查代码
grep -n "Other" 收集表.html

// 2. 发现问题
代码：`${role}Other` → programOther
HTML：`progOther` → 不匹配！

// 3. 修复
const otherSceneIds = {
    'program': 'progOther',
    'planning': 'planOther',
    'art': 'artOther'
};
```

**耗时**：5 分钟定位 + 修复

**发送修复版**：
```
message(action="send", filePath="已修复.html")
```


## 💡 OpenClaw 使用技巧

### 技巧 1：直接说需求，不用管实现

❌ **错误方式**：
```
"帮我写一个 HTML，用 JavaScript 实现动态显示，要响应式..."
```

✅ **正确方式**：
```
"做一个收集表，让团队成员填写 AI 使用情况，生成统一格式文本。"
```

**原理**：AI 会自动选择最优技术方案

---

### 技巧 2：及时反馈，快速迭代

❌ **错误方式**：
```
[沉默 30 分钟] → "不对，重新做"
```

✅ **正确方式**：
```
"格式太简陋" → "不够智能" → "加工具多选" → "有 BUG"
```

**效果**：每轮迭代都有明确方向

---

### 技巧 3：BUG 直接说现象

❌ **错误方式**：
```
"代码有问题，你看看哪里错了"
```

✅ **正确方式**：
```
"只有美术可以点击生成，程序和策划还是不行"
```

**效果**：AI 快速定位问题（ID 不匹配）

---

### 技巧 4：文件直接要，不用问怎么拿

❌ **错误方式**：
```
"文件在哪？我怎么下载？"
```

✅ **正确方式**：
```
"你做完表格直接发我文件。"
```

**效果**：AI 自动发送到聊天窗口

---

### 技巧 5：文档一句话生成

❌ **错误方式**：
```
"写个文档，包括背景、需求、技术实现、测试报告..."
```

✅ **正确方式**：
```
"把我们的聊天整理成需求概况，分享给其他人。"
```

**效果**：AI 自动提取关键信息，生成完整文档
