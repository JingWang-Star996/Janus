1️⃣ 背景与目标

📌 使用场景

作为游戏制作人，我需要 AI 助手帮助管理研发团队，包括：

  • 📋 任务分配与进度跟踪
  • 🐛 BUG 收集与问题汇总
  • 📊 日报/周报自动整理
  • 💬 群内信息分析与决策支持

🎯 核心目标

将 OpenClaw AI 助手接入企业飞书，实现：

  1. 身份识别 - 能区分老板指令和员工汇报
  2. 群聊管理 - 可以加入工作群，读取和发送消息
  3. 权限控制 - 只有授权人员可以触发 AI 行动
  4. 数据安全 - 企业数据不出域，隐私保护
💡 关键洞察 个人飞书 OAuth 适合个人使用，但企业场景必须用企业自建应用，这样才能：
  • 访问企业通讯录
  • 加入企业群聊
  • 使用企业级 API 权限

3️⃣ 创建飞书企业自建应用

1

登录飞书开放平台

访问 https://open.feishu.cn/，使用企业管理员账号登录。

2

创建应用

点击「创建应用」 → 选择「企业自建应用」

  • 应用名称：AI 助理（或你喜欢的名字）
  • 应用图标：随便选一个
  • 应用描述：团队管理助手
3

配置机器人

在应用管理页面，点击左侧「机器人」 → 「创建机器人」

  • 名称：AI 助理
  • 头像：选一个
  • 功能介绍：团队管理、进度跟踪、周报汇总
  • 消息接收模式：选择「接收消息」
4

申请权限

点击左侧「权限管理」 → 「权限配置」，搜索并添加以下权限：

权限名称 权限标识 用途
读取和发送群消息 im:message 核心权限，必须
访问群聊信息 im:chat 获取群成员列表
读取员工信息 contact:contact 识别发送者身份
访问日历 calendar:calendar 可选，日程管理
管理任务 task:task 可选，任务分配
访问云文档 drive:drive 可选，文档管理

点击「申请权限」 → 管理员审批（自己申请自己批）

5

获取凭证

点击左侧「凭证信息」，记录：

App ID: cli_xxxxxxxxxxxxxx App Secret: xxxxxxxxxxxxxxxxxxxxxxxx
⚠️ 安全提醒 App Secret 相当于密码，不要泄露！配置文件建议设置 600 权限。
6

发布应用

点击左侧「版本管理与发布」 → 「创建版本」

  • 版本号：1.0.0
  • 更新说明：初始版本
  • 点击「发布」

4️⃣ 配置 OpenClaw

📝 编辑配置文件

打开 ~/.openclaw/openclaw.json，找到 channels.feishu 部分：

{ "channels": { "feishu": { "enabled": true, "appId": "cli_a93172ef3bfa5cb2", "appSecret": "oi8PM3AyPhPkFsSsW4LJXdzelinkXNZ8", "domain": "feishu", "connectionMode": "websocket", "requireMention": false, "dmPolicy": "allowlist", "groupPolicy": "open", "allowFrom": [ "ou_3833a36af6b901cab181b69ce8b762d1", "ou_7e31a302531d68cf0ce249417e4a1d0c" ], "groupAllowFrom": [] } } }

🔑 关键字段说明

字段 说明 推荐值
appId 飞书应用 ID 从凭证信息获取
appSecret 飞书应用密钥 从凭证信息获取
connectionMode 连接模式 "websocket"（推荐）
requireMention 是否需要@才响应 false（群内免@）
dmPolicy 私聊策略 "allowlist"（白名单）
groupPolicy 群聊策略 "open"（开放）或 "allowlist"
allowFrom 私聊白名单 用户 open_id 列表

🔄 重启网关

配置修改后，重启 OpenClaw 网关：

# 方法 1：使用 CLI openclaw gateway restart # 方法 2：发送信号（热重载） kill -USR1 <gateway_pid>
✅ 验证成功 重启后查看日志，确认没有报错：
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log

5️⃣ 飞书开放平台事件订阅

⚙️ 配置步骤

1

进入事件订阅页面

在飞书开放平台应用管理中，点击左侧「事件订阅」

2

启用事件订阅

开启「启用事件订阅」开关

3

选择 WebSocket 方式

选择「WebSocket 方式」（不需要配置 Request URL）

💡 为什么选 WebSocket？
  • 不需要公网 URL
  • 飞书主动连接 OpenClaw
  • 适合内网部署
  • 配置简单，不易出错
4

订阅事件

搜索并添加以下事件：

im.message.receive_v1 ← 接收消息（必须） im.chat.member.join_v1 ← 成员加入群聊（可选）
5

保存配置

点击「保存」，等待状态变为「已启用」

6️⃣ 常见问题排查

🐛 问题 1：企业飞书发消息不回复

# 查看日志 tail -100 /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log | grep feishu # 典型错误 feishu[default]: sender ou_xxx not in DM allowlist

原因：发送者不在白名单中

解决：在 allowFrom 中添加该用户的 open_id

🐛 问题 2：网关启动报错

# 检查配置语法 cat ~/.openclaw/openclaw.json | python3 -m json.tool # 检查插件安装 ls -la ~/.openclaw/extensions/

常见原因：

  • JSON 格式错误（少了逗号或引号）
  • App ID/Secret 配置错误
  • 插件未安装或版本不匹配

🐛 问题 3：收不到群消息

检查清单：

  1. 机器人是否已加入群聊
  2. groupPolicy 是否设置为 "open" 或配置了 groupAllowFrom
  3. 事件订阅中是否添加了 im.message.receive_v1
  4. 应用是否已发布（不是草稿状态）

🐛 问题 4：权限不足

⚠️ 典型症状 工具调用返回 403 错误，提示权限不足。

解决：

  1. 在飞书开放平台检查权限申请状态
  2. 确认管理员已审批
  3. 重新发布应用版本

🔍 调试技巧

# 实时查看日志 tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log # 过滤飞书相关日志 tail -f /tmp/openclaw/openclaw-*.log | grep -E "feishu|lark" # 检查网关状态 openclaw gateway status # 查看已加载的插件 openclaw gateway status | grep -i plugin

7️⃣ 权限与白名单配置

👥 获取用户 open_id

在飞书群里，发送消息后查看日志：

feishu[default]: received message from ou_7e31a302531d68cf0ce249417e4a1d0c

或使用飞书 API 查询：

# 通过姓名搜索用户 curl -X POST "https://open.feishu.cn/open-apis/contact/v3/users/batch_get_id" \ -H "Authorization: Bearer <access_token>" \ -H "Content-Type: application/json" \ -d '{"user_ids": ["ou_xxx"], "id_type": "open_id"}'

📋 白名单策略建议

场景 dmPolicy groupPolicy allowFrom
仅老板可用 allowlist allowlist [老板 ID]
核心团队可用 allowlist allowlist [核心成员 ID 列表]
全员可用 open open []
老板私聊 + 全员群聊 allowlist open [老板 ID]
💡 最佳实践 建议采用「老板私聊 + 全员群聊」模式：
  • 私聊只有老板能触发 AI 行动
  • 群聊中 AI 可以读取消息、整理汇报
  • 员工汇报不会被误认为指令

8️⃣ 测试验证

✅ 测试清单

测试 1：私聊响应

  1. 在飞书私聊机器人发送「你好」
  2. 检查是否收到回复
  3. 查看日志确认消息接收和发送

测试 2：群聊响应

  1. 将机器人拉入测试群
  2. 在群里@机器人发送消息
  3. 检查是否收到回复

测试 3：身份识别

  1. 用白名单内账号发消息 → 应该响应
  2. 用白名单外账号发消息 → 应该忽略（或仅记录）
  3. 查看日志确认 sender resolved 信息

测试 4：工具调用

  1. 发送指令：「查看我的日程」
  2. 检查是否调用 feishu_calendar_event 工具
  3. 验证返回结果正确

9️⃣ 最佳实践总结

🎯 配置原则

  1. 最小权限 - 只申请必要的权限
  2. 白名单优先 - 默认拒绝，按需授权
  3. WebSocket 模式 - 简单可靠，无需公网
  4. 日志监控 - 定期检查异常

📝 运维建议

  • 定期备份配置文件
  • App Secret 不要提交到版本控制
  • 生产环境用独立的飞书应用
  • 敏感操作（删除、批量发送）需要二次确认

🔒 安全提醒

⚠️ 重要
  • 不要将 App Secret 泄露给他人
  • 定期轮换密钥（建议每季度）
  • 离职员工及时从白名单移除
  • 敏感数据（薪资、绩效）不要让 AI 处理

📈 性能优化

  • 群消息多时用 requireMention: true 减少误触发
  • 大文件处理设置超时
  • 定期清理日志文件
  • 高峰期避免批量操作
