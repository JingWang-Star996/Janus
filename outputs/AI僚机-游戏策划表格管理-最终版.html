AI 做僚机：游戏策划的表格管理新姿势

从权限冲突到自然语言操作，一次技术实操与价值思考

一、配表这件事，为什么这么折磨人？

如果你是游戏策划，或者跟游戏开发沾过边，你一定懂那种感觉。

配表，看似简单的活儿，实际做起来简直是精神消耗战。打开表格，发现同事正在编辑，你被锁在外面；好不容易轮到你，改了两个字段，发现数据对不上，又要来回确认；老板突然要一个紧急数据，你翻遍十几个表，找到眼花；想写个复杂公式，愣是查了半小时文档，最后还是手算……

这些事情单个看都不大，但架不住多、碎、杂。每天被这些琐事打断十几次，心流早就没了。等你真正想静下来思考设计，大脑已经疲惫不堪。

"数据不会说谎，但要学会听数据在说什么。"——而现实是，光是让数据乖乖听话，就已经耗尽了策划的精力。

今天，我想分享一次技术实操经验：如何通过 OpenClaw 打通飞书表格，让 AI 成为你的"僚机"，从此告别那些消耗精神的琐碎操作。

二、技术实操：AI + 飞书表格，怎么打通？

2.1 OpenClaw 是什么？

简单说，OpenClaw 是一个 AI Agent 框架，能让 AI 直接调用各种 API 完成操作。它支持飞书全家桶：文档、表格、多维表格、日历、任务、聊天……你只需要用自然语言发指令，AI 就能帮你执行。

核心价值
不用学复杂工具 → 用自然语言描述需求 → AI 自动调用 API 完成

2.2 实操场景一：跨表引用，一句话搞定

游戏项目中，配表经常分散在多个文件。比如"道具表"在 A 文件，"商店表"在 B 文件，你想在商店表里自动填充道具名称，传统做法是：

  1. 打开道具表，找到道具 ID 和名称的对应关系
  2. 手动复制，或者写 VLOOKUP 公式
  3. 如果道具表更新，再手动同步

这个过程枯燥且容易出错。用 OpenClaw，你只需要说：

"帮我在商店表里加一列，根据道具 ID 从道具表自动匹配道具名称。"

AI 会自动帮你写好跨表引用公式：

=VLOOKUP(A2, IMPORTRANGE("道具表URL", "Sheet1!A:C"), 2, FALSE)

你不需要记 IMPORTRANGE 的语法，不需要手动授权跨表访问，AI 会一步步完成。

2.3 实操场景二：复杂公式，自然语言描述即可

有一次，我需要处理一个组合字段：单元格里有多个值，用竖线分隔，每个值又是"类型*数量"的格式。我要根据类型去另一个表匹配名称，再重新拼接成"名称*数量"的格式。

传统做法？写嵌套公式，查文档，调试半天。

用 OpenClaw，我直接说：

"这个字段格式是'类型1*数量1|类型2*数量2'，帮我根据类型 ID 去另一个表匹配名称，输出'名称1*数量1|名称2*数量2'。"

AI 生成的公式：

=IFERROR(TEXTJOIN("|",TRUE,MAP(SPLIT(B5,"|"),LAMBDA(x,VLOOKUP(VALUE(INDEX(SPLIT(x,"*"),2)),IMPORTRANGE("目标表URL","A:D"),2,FALSE)))),"")

涉及 SPLIT、MAP、TEXTJOIN、VLOOKUP、VALUE、INDEX、IFERROR 等七八个函数嵌套。如果让我自己写，至少半小时。用 AI，一句话，几十秒搞定。

2.4 实操场景三：权限不再是问题

传统表格协作最大的痛点：权限冲突。你想编辑，别人也在编辑，要么被锁住，要么互相覆盖。

用 AI + API 的方式，这个问题迎刃而解。AI 通过 API 操作表格，不占用你的编辑窗口。你可以在群里发指令："帮我把道具 ID 123 的价格改成 100"，AI 直接调用 API 完成，不需要你打开表格。

这意味着什么？
你可以在任何终端（手机、平板、甚至智能音箱）操作表格，不需要安装飞书 App，只需要一个能发消息的界面。

2.5 爽点总结：保护心流，减少精神消耗

总结一下这次实操带来的核心价值：

  1. 快速响应：细碎需求不需要打断当前工作，随时发指令让 AI 处理
  2. 保护心流：不需要频繁切换窗口，大脑可以专注在设计本身
  3. 降低门槛：复杂公式不需要学习，自然语言描述即可
  4. 解放终端：手机、群聊、语音，任何场景都能操作表格

三、价值思考：AI 做僚机，对游戏研发意味着什么？

3.1 传统配表的四大痛点

  • 权限冲突：多人协作，互相锁定，效率低下
  • 流程缺失：没有统一的检查规则，错误容易遗漏
  • 操作繁琐：细碎操作多，精神消耗大
  • 门槛高：复杂公式、跨表引用，学习成本高

这些问题，传统工具很难解决。因为它们不是"功能缺失"，而是"工作方式本身的问题"。

3.2 AI 带来的改变：从"工具升级"到"方式升级"

AI 不是给你一个更好的表格工具，而是改变你和表格的交互方式。

更重要的是，AI 保护了心流。
策划不需要在"思考设计"和"操作表格"之间反复横跳，想到什么直接说，AI 去落地。这种体验就像有个永远在线的助理，不睡觉不抱怨，还不会把你的表改乱。

用 AI 做僚机：

  • 从"手动操作"到"指令驱动"：你说需求，AI 执行
  • 从"学习工具"到"自然语言"：不需要记公式语法
  • 从"单人串行"到"多 AI 并行"：一个指令，多个 AI 同时处理不同任务
游戏开发的未来不是"人 vs AI"，而是"人 + AI vs 问题"。

3.3 对小型团队 / 独立开发者的意义

大公司可以雇专职数据配置，小团队呢？策划既是设计师，又是配置工，还是测试员。一个人干三个人的活，精神力被琐事耗尽，核心设计反而没时间打磨。

AI 做僚机，等于给你配了一个"虚拟助理"。它不领工资，不会累，随时待命。你可以把琐碎的配表工作交给它，自己专注在核心设计上。

一人操作多 AI 落地
这是未来的工作方式。你不再是"一个人干所有事"，而是"一个人指挥多个 AI 完成不同任务"。效率提升不是 2 倍、3 倍，而是质的飞跃。

3.4 未来展望：AI 原生的工作流

这次实操让我看到了一个趋势：AI 原生工作流。

未来的游戏研发，可能不再有"打开表格手动编辑"这个动作。你只需要在群里说一句"把 XX 道具的价格调低 10%"，AI 自动完成，同时更新所有相关数据、生成变更日志、通知到相关人员。

一个主策划可以同时操作三个 AI 僚机——一个管数值平衡，实时模拟调整后的战斗曲线；一个管配置完整性，自动检查漏填和逻辑冲突；一个管版本对比，告诉你这周改了什么、可能影响什么。人做决策，AI 做执行；人负责"为什么"，AI 负责"是什么"和"怎么做"。

这不是替代，是放大。

最终，好的工具让人更像人——少做机器该做的事，多做只有人能做的事：创造、判断、赋予意义。表格还是那个表格，但策划终于可以从表格里抬起头，重新做回"设计游戏"的人。

四、结语：狠狠地用 AI

核心循环不稳，一切都是空中楼阁。配表就是游戏研发的核心循环之一。

如果这个环节还在用传统方式——手动打开表格、复制粘贴、查公式文档、被权限锁住——那你的精神力已经被消耗殆尽了。真正重要的设计工作，反而没精力打磨。

AI 真好用，狠狠地用。

AI 做僚机，你做指挥官。这才是未来的工作方式。
