## 工作流程（强制）⭐

### 阶段一：需求确认（不可跳过）

**必须主动询问**（7 个问题）：

```
1. 主题/核心思想是什么？
2. 文章基调？（哲学思辨 / 行业洞察 / 大众科普）
3. 目标读者？（从业者 / 科技爱好者 / 普通大众 / 自己）
4. 内容侧重？（哲学 / 技术 / 案例 / 平衡）
5. 反驳观点？（是否需要明确反驳某些论调）
6. 人称视角？（第一人称 / 第三人称 / 署名）
7. 输出形式？（全文字数 + 翻页版页数）
```

**示例对话**：
> 用户：帮我写一篇文章
> 
> 助手：好的！先确认几个细节：
> 1. 主题是什么？
> 2. 基调偏向？（深度思考 / 行业分析 / 大众科普）
> 3. 读者是谁？（从业者 / 普通大众）
> 4. 需要反驳某些观点吗？
> 5. 用第一人称还是客观视角？
> 6. 全文多少字？翻页版多少页？

---

### 阶段零：信息收集与思考（新增）⭐

在提问之前，必须先：

1. **阅读用户提供的原始信息**
   - 提取核心思想
   - 识别关键词
   - 理解情感倾向

2. **基于信息思考**
   - 这篇文章的独特价值是什么？
   - 用户真正想表达的是什么？
   - 哪些部分需要补充资料？

3. **整理后提问**
   - 先复述理解的核心思想
   - 再提出针对性问题
   - 给出专业建议供选择

**禁止**：不读信息就直接套模板提问。

---

### 阶段二：大纲确认（不可跳过）⭐

需求确认后，必须先写**文章大纲**给用户确认：

```
1. 文章标题（主标题 + 副标题）
2. 章节结构（3-6 章，每章标题 + 核心要点）
3. 预估字数（每章 800-1000 字）
4. 翻页版页数分配
```

**用户确认后才能开始写作**。

---

### 阶段三：内容创作

#### 3.1 三版本输出（强制）⭐

**必须同时输出三个版本**：

| 版本 | 字数/页数 | 用途 | 格式 |
|------|----------|------|------|
| 全文版 | 3000-5000 字 | 深度阅读、存档 | HTML 长文 |
| 翻页版 | 最多 18 页 | 手机分享、点击翻页 | HTML 分页 |
| 截图版 | 9 页（每页 2 屏） | 滚动截图、拼图分享 | HTML 长页 |

#### 3.2 结构规划

**全文版结构**（根据主题推荐）：
- 引言（10%）：引入话题、表明立场
- 主体（70%）：3-5 个章节，层层递进
- 结语（20%）：升华主题、行动号召

**翻页版结构**：
```
封面页（大字标题）→ 引言 → 主体 (12-15 页) → 结语 → 尾页
```
- 每页高度：667px（一屏）
- 每页 200-300 字
- 点击翻页 + 触摸滑动

**截图版结构**：
```
封面页 → 内容 (7 页) → 尾页
```
- 每页高度：1334px（两屏，可滚动）
- 每页 400-600 字
- 用户自己滚动截图，方便拼图

#### 3.3 人称与身份（重要）⭐

根据用户需求确定：

| 要素 | 选项 | 示例 |
|------|------|------|
| 人称 | 第一人称 | "我认为"、"我观察到" |
| 身份 | 职业/角色 | "游戏人王鲸"、"游戏制作人" |
| 署名 | 是/否 | 文末署名"jing"或笔名 |

**第一人称写作要点**：
- ✅ 表达个人观点（"我不这么认为"）
- ✅ 分享个人经历（"我观察到"）
- ✅ 表明身份立场（"作为游戏制作人"）
- ❌ 避免过度主观（保持论证严谨）

---

#### 3.4 内容要求

| 元素 | 使用场景 | 样式 |
|------|---------|------|
| 数据框 | 数据、指标、对比 | 深色背景 #1a1a2e |
| 引用框 | 金句、引言 | 渐变粉色 #f093fb→#f5576c |
| 高亮框 | 核心观点 | 渐变紫色 #667eea→#764ba2 |
| 列表 | 要点罗列 | 有序/无序 |
| 表格 | 对比数据 | 简洁边框 |

### 阶段四：HTML 生成

#### 4.1 三版本生成（强制）⭐

**全文版 HTML**：
- 单页长文，适合桌面阅读
- 清晰章节导航
- 可打印/存档

**翻页版 HTML**：
- 移动端优先（375px 宽度）
- 每页 667px 高度（一屏）
- 翻页导航 + 触摸滑动
- 最多 18 页（含封面）

**截图版 HTML**：
- 移动端优先（375px 宽度）
- 每页 1334px 高度（两屏）
- 无导航按钮（方便滚动截图）
- 固定 9 页（含封面）
- 每页底部有页码标识

#### 4.2 页面规范

**移动端优先**：
- 宽度：375px（iPhone 标准）
- 字体：PingFang SC, Microsoft YaHei

**翻页版必要组件**：
1. ✅ 封面页（大标题 + 副标题 + 金句）
2. ✅ 分页指示器（圆点）
3. ✅ 导航按钮（上一页/下一页）
4. ✅ 触摸滑动支持
5. ✅ 页码显示
6. ✅ 底部水印

**截图版规范**：
- ❌ 无导航按钮（不遮挡内容）
- ✅ 每页底部页码（第 X 页/共 9 页）
- ✅ 每页 400-600 字

#### 3.2 样式组件库

/* 引用框 */
.quote-box {
    background: linear-gradient(135deg, #f093fb 0%, #f5576c 100%);
    color: #fff;
    padding: 16px 20px;
    border-radius: 12px;
    font-style: italic;

/* 高亮框 */
.highlight {
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    color: #fff;
    padding: 16px 20px;

### 阶段四：上传与发送（强制）⭐

**步骤**：

1. **保存文件**
   ```
   路径：/home/z3129119/.openclaw/workspace/outputs/[主题]-[日期].html
   命名：kebab-case，含日期
   ```

2. **上传飞书云空间**
   ```
   工具：feishu_drive_file upload
   参数：file_path=[HTML 文件路径]
   ```

3. **获取文件链接**
   ```
   工具：feishu_drive_file list
   提取：最新文件的 url 字段
   ```

4. **发送给用户（必须立即执行！）⭐**
   ```
   在回复中立即发送链接，格式：
   
   **📄 全文版**：[链接]
   **📱 翻页版**：[链接]
   **📸 截图版**：[链接]
   ```

**检查清单**：
- [ ] HTML 文件已保存
- [ ] 已上传飞书云空间
- [ ] **从飞书云空间获取最新文件链接**（不要使用旧链接）
- [ ] **已立即在回复中发送链接给用户**（不要等用户催促）
- [ ] 说明包含：文件名、链接、特点

**重要**：
1. 上传后必须立即从云空间获取最新文件列表，提取正确的 file token 和 url
2. **禁止使用之前缓存的链接**
3. **禁止"以为发了"但实际没发** → 必须在回复中明确显示链接
4. **不要等用户催促才发链接** → 上传完成后立即发送

## 错误预防清单 ⭐

### 常见错误及避免方法

| 错误 | 原因 | 避免方法 |
|------|------|---------|
| 忘记发送文件 | 创建后未主动发送 | 阶段四为强制步骤 |
| 格式不符合规范 | 未读技能文档 | 写作前必读 SKILL.md |
| 缺少封面/分页 | 凭经验写作 | 按阶段二结构规划 |
| 授权失败 | 用户身份工具需 OAuth | 失败后立即引导授权 |
| 内容过长 | 未控制每页字数 | 每页 200-400 字上限 |

4. [ ] 每页 200-400 字吗？
