# 王鲸 AI-创作工作流程

**场景**：从对话到发布的完整流程

---

## 📋 完整流程

```
1. 对话交流
   ↓
2. 提取关键信息
   ↓
3. 更新画像（如有新认知）
   ↓
4. 创作文案
   ↓
5. 审查文案（是否符合风格）
   ↓
6. 用户确认/修改
   ↓
7. 发布
   ↓
8. 记录反馈（用于优化画像）
```

---

## 🎯 场景 1：朋友圈发布

### 步骤 1：对话交流
```
用户：Claude 源码泄露第二天我就实战了
```

### 步骤 2：提取关键信息
- 技术嗅觉敏锐（24 小时跟进）
- 快速落地能力（实战，不是纸上谈兵）

### 步骤 3：更新画像
```markdown
## 2026-04-02 更新

**来源**：朋友圈文案讨论

**新认知**：
- Claude 源码泄露第二天就实战
- 强调"快"是核心竞争力
```

### 步骤 4：创作文案
```
调用创作 Skill，基于画像生成文案
```

### 步骤 5：审查文案
```
检查：
- 是否体现了"快"？✅
- 技术细节精准吗？✅
- 语气从容自信吗？✅
- 有行业洞察吗？✅
```

### 步骤 6：用户确认
```
用户：可以 / 需要修改 XXX
```

### 步骤 7：发布
```
复制文案 + 配图 → 朋友圈
```

### 步骤 8：记录反馈
```
用户：这个文案效果不错 / 不太满意

→ 更新画像：用户喜欢/讨厌这种风格
```

## 🎯 场景 3：紧急公关

**场景**：发错群、说错话、需要澄清

### 快速响应流程
```
1. 立即道歉（真诚，不找借口）
2. 说明原因（简短，不啰嗦）
3. 补救措施（已做什么）
4. 预防机制（以后怎么避免）
```

### 示例（发错群）
```
抱歉！发错群了😅

这是内部通知，不应该发到孵化大群。

已发澄清消息："抱歉，刚才的消息发错群了，请忽略～"

已建立审查流程：
- 发送前判断消息类型
- 内部通知→私聊
- 工作安排→群聊

以后不会再犯。
```

## 📊 文案创作检查清单

### 朋友圈
- [ ] 有具体细节吗？（不是空泛形容词）
- [ ] 有洞察总结吗？（给读者价值）
- [ ] 语气从容吗？（不装逼、不硬广）
- [ ] 有开放姿态吗？（欢迎合作/交流）
- [ ] 符合老板身份吗？（不卑微、不炫耀）
- [ ] 技术关键词精准吗？（证明真懂）
- [ ] 长度适中有吗？（300-500 字最佳）

### 小红书
- [ ] 标题抓眼球吗？（数字/痛点/好奇心）
- [ ] 开篇真诚吗？（不装逼、不硬广）
- [ ] 痛点描述精准吗？（引发共鸣）
- [ ] 技术细节有吗？（证明真懂）
- [ ] 场景举例有吗？（让抽象变具体）
- [ ] 互动引导有吗？（评论区活跃）
- [ ] 标签合适吗？（3-5 个相关标签）
