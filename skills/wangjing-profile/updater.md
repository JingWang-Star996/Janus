# 王鲸 AI-画像更新机制

**目的**：持续学习王鲸的特质、偏好、风格，让创作越来越像"本人"

---

## 🔄 更新触发条件

### 1. 用户明确要求
- "记住这个"
- "把这个写进画像"
- "以后按这个风格来"

### 2. 用户纠正风格
- "这个不像我"
- "我不会这么说"
- "太装了，我不是这样的"

### 3. 用户展现新特质
- 新的工作方式
- 新的价值观表达
- 新的语言习惯

### 4. 用户强烈反应
- 特别喜欢某文案
- 特别讨厌某表达
- 反复强调某观点

---

## 📝 更新流程

```
对话中检测到新认知
    ↓
提取关键信息（特质/偏好/风格）
    ↓
归类到画像维度
    ↓
写入 profile.md
    ↓
通知用户"画像已更新"
```

---

## 🧠 更新示例

### 示例 1：价值观更新

**对话**：
```
用户：我对抄袭不羞耻，我展示的是跟随 AI 热点和商用落地的能力
```

**更新**：
```markdown
## 2026-04-02 更新

**来源**：朋友圈文案创作对话

**新认知**：
- 对"抄袭"不羞耻，强调快速落地能力
- 公开承认基于开源框架，不包装成原创

**更新维度**：
- 价值观→对原创/抄袭的看法
```

---

### 示例 2：场景策略更新

**对话**：
```
用户：朋友圈有同事、老板、潜在投资人，不用避开，能力展示也是一环
```

**更新**：
```markdown
## 2026-04-02 更新

**来源**：朋友圈文案策略讨论

**新认知**：
- 朋友圈受众包含同事/下属/投资人/潜在合作者
- 不避开，反而主动展示能力
- 目的是吸引合作、建立影响力

**更新维度**：
- 社交场景→朋友圈策略
```

---

### 示例 3：语言风格更新

**对话**：
```
用户：不要"说不上多厉害"这种过度谦虚，假
```

**更新**：
```markdown
## 2026-04-02 更新

**来源**：文案审查反馈

**新认知**：
- 讨厌过度谦虚的表达
- 认为"说不上多厉害"这种话假

**更新维度**：
- 语言风格→禁忌表达
- 新增："说不上多厉害"、"还比较基础"（过度谦虚）
```

## 📊 画像版本管理

### 版本记录
每次更新时，在 profile.md 末尾追加：

```markdown
## 📝 更新日志

| 日期 | 更新内容 | 来源 |
|------|---------|------|
| 2026-04-02 20:48 | 初始版本，完整画像 | 朋友圈文案创作对话 |
| 2026-04-02 21:00 | 添加小红书示例库 | 技能完善 |
```

### 备份机制
每次更新前自动备份：
```bash
cp profile.md profile.md.backup-$(date +%Y%m%d-%H%M%S)
```

## 🎯 自动检测逻辑

### 关键词检测
```javascript
const updateTriggers = [
  '记住这个',
  '把这个写进',
  '以后按这个',
  '我不是这样的',
  '这个不像我',
  '我讨厌',
  '我喜欢',
  '我一直觉得',
  '我的原则是'
];
```

### 情感强度检测
- 感叹号（！/!）→ 强烈情感
- 重复强调 → 重要观点
- 否定句（"不是...是..."）→ 价值观表达

---

## 📋 更新检查清单

每次更新前自查：

- [ ] 这个认知是新的吗？（不是重复已知）
- [ ] 有具体来源吗？（能追溯到对话）
- [ ] 归类准确吗？（放到合适维度）
- [ ] 表述清晰吗？（其他人能看懂）
- [ ] 需要备份吗？（重要更新先备份）
