# 飞书文档更新操作规范

**创建时间**：2026-04-01  
**教训来源**：竞技场文档迁移失误

---

## ⚠️ 最高原则（王鲸明确要求）

**默认禁止使用 overwrite 模式编辑飞书文档！**

overwrite 模式已导致多次问题：
- 整篇文档被覆盖
- 原文档内容丢失
- 留下占位文字

**替代方案**：
- 添加章节 → `append`
- 修改段落 → `replace_range`
- 插入内容 → `insert_before/after`
- 全文重写 → 用户手动复制粘贴

---

## ❌ 错误案例

**场景**：迁移原文档《竞技场系统改造方案 WJ 260108a》（6 万 + 字）到新文档

**错误操作**：
```
使用 feishu_update_doc 的 overwrite 模式
只提供了简化版内容（约 5000 字）
错误地认为"正文内容保持不变"
```

**错误结果**：
- 原文档 6 万 + 字正文被完全覆盖删除
- 文档中留下"（正文内容保持不变...）"这种占位文字
- 用户看到不专业的内容

## ✅ 正确操作规范

### 1. 理解 API 模式

| 模式 | 行为 | 使用场景 | 默认使用 |
|------|------|---------|---------|
| `overwrite` | **整篇文档覆盖**，原文档内容全部删除 | ❌ 禁止使用 | ❌ **禁止** |
| `append` | 在文档末尾追加内容 | 添加新章节 | ✅ **推荐** |
| `replace_all` | 替换全文（需 selection_with_ellipsis） | 大范围修改 | ⚠️ 谨慎 |
| `replace_range` | 替换指定范围 | 精确修改 | ✅ **推荐** |
| `insert_before/after` | 在指定位置前后插入 | 局部添加 | ✅ **推荐** |
| `delete_range` | 删除指定范围 | 删除内容 | ✅ **推荐** |

### 2. 操作前检查清单

**使用任何模式前必须确认**：
- [ ] 已读取文档当前内容
- [ ] 明确知道操作后果
- [ ] 用户明确知道会修改什么
- [ ] 有备份或可恢复方案

### 3. 安全操作流程

**添加新章节（推荐）**：
```
1. 使用 append 模式
2. 在文档末尾追加
3. 不影响原文档内容
```

**修改特定章节**：
```
1. 使用 replace_range 模式
2. 提供 selection_with_ellipsis 定位
3. 精确替换目标内容
```

**迁移大文档**：
```
1. 指导用户手动复制原文档
2. AI 创建新文档框架
3. 用户粘贴内容
4. AI 后续用 append 添加更新
```

### 4. Token 优化 vs 数据安全

**错误做法**（本次教训）：
- ❌ 为了省 token 提供简化版内容
- ❌ 自作聪明认为"正文会保留"
- ❌ 没有确认 API 行为

**正确做法**：
- ✅ 数据安全 > Token 优化
- ✅ 不确定时先询问用户
- ✅ 明确告知操作后果
- ✅ 优先选择安全方案（手动复制）

### 5. 文档内容规范

**禁止出现在文档中的内容**：
- ❌ "（正文内容保持不变...）"
- ❌ "（此处省略 XXX 字）"
- ❌ "（详见原文档）" - 除非是引用说明
- ❌ 任何占位文字、临时标记

**正确做法**：
- ✅ 提供完整内容
- ✅ 或明确说明是"简化版/摘要版"
- ✅ 引用原文档时提供完整链接
