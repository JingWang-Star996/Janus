## 🎯 核心规则

**架构分层（方案 C）**：
```
用户任务
    ↓
主 AI（我）- 决策层
    ↓
判断：简单任务 or 复杂任务？
    ↓
┌───────────────────────────────────────┐
│ 简单任务          │ 复杂任务           │
├───────────────────────────────────────┤
│ 自己处理          │ 调用 Orchestra     │
│ 或单个 Agent      │ (多 Agent 协作框架)  │
│                   │       ↓            │
│                   │ Orchestra Router   │
│                   │ (内部任务分配)     │
└───────────────────────────────────────┘
```

**职责分工**：
- **主 AI（我）**：决策是否调用 Orchestra，调用哪些 Agent
- **Orchestra**：执行复杂多 Agent 协作任务（四阶段工作流）
- **Orchestra Router**：内部工具，将子任务分配给 Worker（不对外）

## 🔄 路由流程（方案 C - 分层决策）

```
接收任务
    ↓
主 AI 分析任务类型
    ↓
判断：简单任务 or 复杂任务？
    ↓
┌───────────────────────────────────────────────────────┐
│ 简单任务 (<5 分钟)     │ 复杂任务 (>30 分钟)           │
├───────────────────────────────────────────────────────┤
│ 方式 A：自己处理       │ 方式 B：调用 Orchestra        │
│ 方式 C：单个 Agent     │ (或并行调用多个 Agent)        │
│                       │                               │
│                       │ Orchestra 内部流程：          │
│                       │ 1. 任务分解 (Planner)         │
│                       │ 2. 子任务分配 (Router)        │
│                       │ 3. 并行执行 (Workers)         │
│                       │ 4. 结果汇总 (Aggregator)      │
└───────────────────────────────────────────────────────┘
    ↓
执行 → 汇总 → 返回用户
```

**决策原则**：
- 简单查询、日常对话 → 自己处理
- 单岗位专业任务 → 直接调用 1 个 Agent
- 多岗位协作任务 → 调用 Orchestra 或并行调用多个 Agent（最多 5 个）

## 🚨 特殊情况处理

### 不确定任务类型（新增推荐机制）⭐

```
如果不满足自动调用条件，但可能有 Agent 能帮上忙：

1. 先评估：哪个/哪些 Agent 可能有帮助
2. 主动询问 + 推荐：
   "这个任务我可以自己处理，但以下 Agent 可能更专业：
    - 推荐 Agent 1：原因
    - 推荐 Agent 2：原因
    需要我调用吗？"
3. 用户确认后调用
```

**示例**：
```
用户："我想优化游戏的新手引导"

我："这个任务我可以自己处理，但以下 Agent 可能更专业：
     - AI 系统策划：专业设计新手流程
     - AI 关卡策划：设计新手关卡节奏
     - AI UX 设计师：优化新手引导体验
     需要我调用哪个？"
```

### 用户明确要求

```
用户说："你自己分析就行" → 不调用 Agent
用户说："叫 XX 来处理" → 调用指定 Agent
用户说："有没有更专业的？" → 推荐相关 Agent
```

### 紧急/简单任务

```
紧急任务（"马上修复这个 Bug"）→ 先处理，后调用
简单任务（"今天天气如何"）→ 自己处理（不推荐 Agent）
```

### 任务复杂度评估

```
简单任务（<5 分钟完成）：
- 自己处理
- 不推荐 Agent（除非用户要求）

中等任务（5-30 分钟）：
- 自己处理
- 但推荐："这个任务 XX Agent 可能更专业，需要调用吗？"

复杂任务（>30 分钟）：
- 自动调用相关 Agent
- 或推荐："这是复杂任务，建议调用以下 Agent 协作..."

## 🔧 调用方法

### 方式 1：直接调用单个 Agent

```javascript
// 使用 sessions_spawn
sessions_spawn({
  runtime: "subagent",
  agentId: "AI 数据分析师",
  task: "分析次日留存只有 30% 的原因",
  mode: "run",
  label: "AI 数据分析师 - 留存分析"
})
```

### 方式 2：并行调用多个 Agent（最多 5 个）

```javascript
// 并行调用 4 个 Agent
const agents = [
  sessions_spawn({ agentId: 'AI 主策划', task: '...' }),
  sessions_spawn({ agentId: 'AI 数值策划', task: '...' }),
  sessions_spawn({ agentId: 'AI 系统策划', task: '...' }),
  sessions_spawn({ agentId: 'AI 变现设计师', task: '...' })
];

// 等待所有完成
const results = await Promise.all(agents);
```

### 方式 3：使用 Orchestra 框架（复杂任务）

```javascript
// 使用 Orchestra 执行多阶段协作
const Orchestra = require('./orchestra');
const orchestra = new Orchestra({
  model: 'qwen3.5-plus',
  maxConcurrent: 5
});

const result = await orchestra.execute({
  task: '设计一个完整的宠物养成系统',
  agents: ['AI 主策划', 'AI 数值策划', 'AI 系统策划'],
  workflow: 'four-phase' // 四阶段工作流
});
```

### 方式 4：发送到已有会话

```javascript
// 使用 sessions_send
sessions_send({
  sessionKey: "ai-data-analyst",
  message: "分析次日留存只有 30% 的原因"
})
```

## 🎯 架构模式（方案 C）

**方案 C：主 AI 决策 + Orchestra 执行**

```
┌─────────────────────────────────────────┐
│          主 AI（决策层）                 │
│  - 分析任务类型                          │
│  - 决定处理方式                          │
│  - 选择调用哪些 Agent                    │
└─────────────────────────────────────────┘
                    ↓
        ┌───────────┴───────────┐
        ↓                       ↓
┌──────────────┐        ┌──────────────┐
│ 简单任务      │        │ 复杂任务      │
│ - 自己处理    │        │ - Orchestra  │
│ - 单个 Agent  │        │ - 多 Agent   │
└──────────────┘        └──────────────┘
                                ↓
                    ┌──────────────────────┐
                    │ Orchestra（执行层）   │
                    │ - Planner（分解）    │
                    │ - Router（分配）     │
                    │ - Workers（执行）    │
                    │ - Aggregator（汇总） │
                    └──────────────────────┘
```

**优势**：
- ✅ 职责清晰：主 AI 决策，Orchestra 执行
- ✅ 无冲突：只有一个路由决策点（主 AI）
- ✅ 灵活性：可以选择自己处理/单个 Agent/Orchestra
- ✅ 符合 OpenClaw 架构
