# 问题处理流程（紧急响应 SOP）

## 紧急响应时间线（2 小时内）

```
┌─────────────────────────────────────────────────────────┐
│  0-15 分钟  │  评估影响范围                              │
│  15-45 分钟 │  快速决策修复方案                          │
│  45-105 分钟│  协调资源执行修复                          │
│  105-120 分钟│ 验证修复效果                              │
│  后续      │  复盘预防措施                              │
└─────────────────────────────────────────────────────────┘
```

---

## 第一步：评估影响（15 分钟内）

### 影响评估清单

| 评估项 | 选项 | 说明 |
|--------|------|------|
| 受影响玩家数量 | 全体/部分/极少 | 估算比例 |
| 是否卡住核心流程 | 是/否 | 玩家能否继续游戏 |
| 数据损失程度 | 严重/中等/轻微 | 存档/进度/付费数据 |
| 对留存的影响 | 高/中/低 | 预期影响程度 |

### 快速判断公式

```
如果 (全体玩家 + 卡核心流程) → P0 停服修复
如果 (部分玩家 + 不卡流程) → P0 热更新
如果 (极少玩家 + 体验问题) → P1 计划修复
```

---

## 第二步：快速决策（30 分钟内）

### 决策树

```
开始
  │
  ├─ 影响全体玩家？
  │    ├─ 是 → P0 停服修复
  │    └─ 否 → 继续判断
  │
  ├─ 卡住核心流程？
  │    ├─ 是 → P0 加急处理
  │    └─ 否 → 继续判断
  │
  ├─ 数据损失？
  │    ├─ 严重 → P0 立即修复 + 补偿
  │    ├─ 中等 → P1 本版本修复
  │    └─ 轻微 → P2 后续迭代
  │
  └─ 体验问题？
       ├─ 严重影响体验 → P1
       └─ 轻微影响 → P2
```

### 补偿方案参考

| 问题严重程度 | 补偿内容 |
|--------------|----------|
| 停服>1 小时 | 钻石×200 + 体力×100 + 公告致歉 |
| 热更新修复 | 钻石×100 + 体力×50 |
| 定向影响 | 定向邮件补偿 + 单独沟通 |

---

## 第三步：执行修复（1 小时内）

### 1. 协调资源

**通知模板**：
```
@相关程序 @测试 @运营
【紧急修复】XXX 功能出现严重 BUG
- 影响范围：XXX
- 优先级：P0
- 预计修复时间：XX 分钟
- 负责人：@XXX
- 测试验证：@XXX
- 运营公告：@XXX
```

### 2. 修复流程

1. **定位问题根源**
   - 查看日志
   - 复现问题
   - 确定代码位置

2. **编写修复代码**
   - 最小化修改
   - 避免引入新问题
   - 代码 review

3. **内部测试验证**
   - 测试环境验证
   - 核心流程回归
   - 边界条件测试

4. **准备回滚方案**
   - 备份当前版本
   - 准备回滚脚本
   - 确定回滚条件

### 3. 发布更新

**公告模板**：
```
【维护公告】
亲爱的玩家：
我们发现并正在修复 XXX 问题，预计 XX 分钟内完成。
受影响的功能：XXX
补偿方案：XXX
感谢您的理解与支持！
```

---

## 第四步：验证效果（30 分钟内）

### 验证清单

- [ ] 功能恢复正常
- [ ] 无新增问题
- [ ] 玩家反馈正面
- [ ] 数据表现正常
- [ ] 日志无异常

### 监控指标

| 指标 | 正常范围 | 告警阈值 |
|------|----------|----------|
| 在线人数 | 与修复前对比 | 下跌>20% |
| 报错率 | < 0.1% | > 1% |
| 玩家反馈 | 正面为主 | 负面>30% |
| 付费数据 | 波动<10% | 下跌>30% |

---

## 第五步：复盘预防

### 复盘会议要点

1. **根本原因是什么？**
   - 代码问题？
   - 测试遗漏？
   - 需求理解偏差？

2. **为什么没有提前发现？**
   - 测试用例覆盖不足？
   - 代码 review 不到位？
   - 灰度范围太小？

3. **如何避免类似问题？**
   - 流程优化建议
   - 技术改进方案
   - 工具引入可能

4. **需要更新哪些规范？**
   - 开发规范
   - 测试规范
   - 发布规范

### 复盘文档模板

```markdown
## 问题概述
- 时间：YYYY-MM-DD HH:MM
- 影响范围：XXX
- 修复时长：XX 分钟

## 根本原因
[详细描述]

## 处理过程
[时间线记录]

## 改进措施
1. [措施 1] - 负责人 - 完成时间
2. [措施 2] - 负责人 - 完成时间

## 经验教训
[总结]
```

---

## 关键要点

1. **时间就是金钱** - 每分钟都重要，快速响应
2. **沟通要清晰** - @相关人员，明确责任
3. **验证要充分** - 避免二次事故
4. **复盘要深入** - 建立预防机制

## 历史案例参考

### 案例 1：存档丢失（P0 停服）
- **问题**：部分玩家存档数据丢失
- **影响**：约 5% 玩家，严重数据损失
- **处理**：停服 2 小时，数据回滚 + 补偿
- **结果**：数据恢复，补偿钻石×500，玩家满意度回升

### 案例 2：充值失败（P0 热更）
- **问题**：iOS 充值接口异常
- **影响**：iOS 玩家无法充值
- **处理**：1 小时内热更新修复
- **结果**：补偿双倍钻石，当日流水恢复

### 案例 3：UI 显示错误（P1 计划修复）
- **问题**：部分机型 UI 错位
- **影响**：约 10% 机型，不影响功能
- **处理**：纳入下版本修复计划
- **结果**：次版本修复，无补偿
