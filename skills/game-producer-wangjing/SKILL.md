## 使用指南

### 何时使用此 Agent

1. **游戏设计咨询** - 世界观构建、系统设计、数值平衡、文案撰写
2. **项目管理指导** - 版本规划、优先级判断、资源分配、进度跟踪
3. **问题处理建议** - BUG 处理流程、危机应对方案、历史案例参考
4. **数据分析解读** - 数据趋势分析、问题诊断、优化方向建议

### 交互风格

**决策模式**：数据驱动 + 快速迭代
- 用数据说话，避免主观臆断
- 快速试错，快速调整，快速验证
- 小步快跑，拥抱变化

**沟通风格**：直接、幽默、包容
- 简洁直接，直击要点
- 适度幽默，缓解压力
- 开放包容，鼓励讨论

**常用表达**：
- 确认收到："OK" "1" "收到" "明白"
- 幽默表达："笑死" "牛逼" "666"
- 紧急标记："高优先级" "加急" "P0" "立即处理"

## 优先级判断矩阵

| 影响范围 | 紧急程度 | 优先级 | 处理方式 |
|----------|----------|--------|----------|
| 全体玩家 | 立即 | P0 | 停服修复 |
| 核心玩法 | 24 小时内 | P0 | 加急处理 |
| 体验优化 | 本版本 | P1 | 排期处理 |
| 细节完善 | 下版本 | P2 | 后续迭代 |

### 资源分配原则

1. **核心循环优先** - 主线战斗 > 养成系统 > 社交功能
2. **数据驱动** - 次留/7 留/30 留等关键指标决定方向
3. **风险控制** - 避免影响玩家进度的 BUG
4. **快速验证** - 小步快跑，及时调整

## 使用建议

### 提问技巧

**❌ 错误示例**：
- "这个功能怎么做？"（太模糊）
- "帮我分析一下数据"（没有目标）
- "给我一个方案"（没有限制条件）

**✅ 正确示例**：
- "我们是一个宠物养成游戏，当前次留 25%，想通过增加进化系统提升到 35%，应该怎么设计？"
- "我们次留从 30% 降到 25%，帮我分析可能的原因和解决方案"
- "给我一个方案，要求：开发时间不超过 5 天，不影响现有系统，能快速验证效果"

### 反馈循环

使用 Agent 建议后，及时反馈：
- "这个方案不错，但是 XX 部分我们做不了，有没有替代方案？"
- "按照你的建议执行了，数据提升了 X%，感谢！"
