# work-log-review - 工作日志复盘与原因分析 Skill

**版本**：1.0  
**创建时间**：2026-03-28  
**触发场景**：用户指出遗漏、错误、修复问题时

---

## 🎯 核心原则

**当用户指出任何遗漏、错误、需要修复的问题时，必须执行以下流程：**

```
发现问题 → 立即修复 → 复盘分析 → 记录原因 → 写成 Skill → 更新记忆
```

**禁止行为**：
- ❌ 只修复不复盘
- ❌ 只道歉不分析原因
- ❌ 同样的错误犯两次
- ❌ 不更新 Skill 和记忆

---

## 📋 执行流程

### 步骤 1：立即修复（优先级：最高）

**动作**：
1. 第一时间修复用户指出的问题
2. 确认修复结果
3. 向用户确认是否完成

**示例**：
```
用户："你漏了 XX 工作"
→ 立即补充遗漏内容
→ 更新文档/文件
→ 发送链接确认
```

---

### 步骤 2：复盘分析（必须执行）

**回答以下问题**：

1. **为什么会漏掉？**
   - 信息获取不完整？
   - 理解偏差？
   - 优先级判断错误？
   - 记忆/上下文丢失？
   - 工具/流程限制？

2. **问题出在哪个环节？**
   - 信息收集阶段？
   - 信息处理阶段？
   - 信息输出阶段？
   - 记忆存储阶段？

3. **如何避免再次发生？**
   - 需要增加什么检查步骤？
   - 需要修改什么流程？
   - 需要创建什么 Skill？
   - 需要更新什么记忆？

---

## 🧠 本次案例分析（2026-03-28）

### 问题：遗漏重要工作内容

**发现时间**：2026-03-28 15:11

**用户反馈**：
> "你漏了工作吧，昨天不是还让你参加了一场会议，并且让你用全公司出方案，以及让你和真人员工对接创作方案"

### 原因分析

1. **信息获取不完整**
   - 仅依赖 MEMORY.md 和 memory/目录查找
   - 没有主动搜索所有相关文件
   - 没有检查 agents/目录创建记录

2. **理解偏差**
   - 认为"昨天的工作"仅指系统记录的任务
   - 忽略了实际执行的会议和方案创作工作
   - 没有理解用户要求的"所有操作和事情"

3. **记忆存储缺失**
   - 会议和方案创作没有写入 memory/2026-03-27.md
   - 重要工作仅存在于对话上下文中
   - 没有在任务完成后立即记录

4. **检查流程不完善**
   - 没有主动询问用户"是否有遗漏"
   - 没有检查飞书文档创建记录
   - 没有检查 agents 目录创建时间

### 根本原因

**没有建立"工作完成→立即记录→定期复盘"的闭环流程**

### 解决方案

**短期方案**（立即执行）：
1. ✅ 立即补充遗漏内容到日报
2. ✅ 创建 work-log-review Skill
3. ✅ 更新 MEMORY.md 记录此原则

**长期方案**（流程改进）：
1. ✅ 每次重要工作完成后立即写入 memory/YYYY-MM-DD.md
2. ✅ 创建/修改文件时自动记录到工作日志
3. ✅ 输出报告前主动询问用户"是否有遗漏"
4. ✅ 定期检查 agents/、docs/、outputs/目录变更

### 预防措施

```
每次输出工作报告前执行检查清单：
□ 检查 memory/YYYY-MM-DD.md
□ 检查 MEMORY.md 最近记录
□ 检查 agents/目录变更（find -mtime -2）
□ 检查 docs/目录变更
□ 检查 outputs/目录变更
□ 检查飞书文档创建记录
□ 主动询问用户"是否有遗漏的工作"
□ 等待用户确认后再输出最终报告
```
