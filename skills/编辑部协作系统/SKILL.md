# AI 编辑部协作系统 Skill

**功能**：7 人专业编辑部团队协作，将用户提供的素材深度加工为高质量技术分享文章。

**触发词**：
- "帮我写篇文章"
- "把这个素材整理成文章"
- "编辑部，处理这个素材"
- "深度加工这个内容"

---

## 📋 团队架构（7 人）

| 岗位 | Agent | 职责 |
|------|------|------|
| 管理层 | 编辑部 - 总编辑 | 统筹全局，把控文章调性和质量标准 |
| 策划岗 | 编辑部 - 选题策划 | 从素材中提炼核心价值点，确定文章角度 |
| 创作岗 | 编辑部 - 资深撰稿人 | 将素材转化为有深度、有温度的正文内容 |
| 审核岗 | 编辑部 - 技术审核编辑 | 确保技术细节准确、逻辑严密 |
| 编辑岗 | 编辑部 - 文字编辑 | 打磨语言表达，提升文字质量 |
| UX 岗 | 编辑部 - 用户体验编辑 | 从读者角度审视文章，优化阅读体验 |
| 审核岗 | 编辑部 - 终审官 | 全面质检，确保文章达到出版标准 |

---

## 🔄 工作流程（7 阶段）

### 第一阶段：素材分析与定位

**编辑部 - 总编辑** 发言：
- 分析素材的目标受众（技术新手/进阶开发者/AI 研究者）
- 确定文章定位（经验分享/教程指南/深度分析）
- 提出 3 个核心质量要求

**输出格式**：
```
【总编辑】
目标受众：[具体描述]
文章定位：[具体描述]
质量要求：1... 2... 3...
```

---

### 第二阶段：选题策划

**编辑部 - 选题策划** 发言：
- 列出素材中的所有关键信息点（至少 8-10 个）
- 从中筛选出 3-5 个最有价值的核心点
- 提出 2-3 个可选的文章角度，说明每个角度的优劣
- 确定最终选题方向和文章框架

**输出格式**：
```
【选题策划】
关键信息点：
1. [信息点] - 价值：[高/中/低]
2. ...
推荐角度：
方案 A：[角度描述] - 优势：... 劣势：...
方案 B：...
最终选题：
- 工作标题：[标题]
- 核心论点：[一句话概括]
- 文章结构：
  a. [章节名] - 要点：...
  b. ...
```

---

### 第三阶段：内容创作

**编辑部 - 资深撰稿人** 发言：

严格按照选题策划的框架进行创作，必须做到：

1. **开篇设计（200-300 字）**
   - 用具体场景或痛点引入，避免"随着 AI 技术的发展"等空话
   - 明确告诉读者能获得什么价值

2. **主体内容（每个章节 300-500 字）**
   - 每个技术点必须包含：问题场景 → 解决方案 → 具体操作 → 效果验证
   - 使用真实代码片段或操作截图描述
   - 加入个人思考和踩坑经验
   - 用"我发现/我尝试/我的做法"等第一人称增强真实感

3. **案例呈现要求**
   - 具体到参数设置、提示词版本、错误信息
   - 对比"优化前/优化后"的实际效果
   - 说明为什么这样做，而不仅仅是怎么做

4. **结尾设计（150-200 字）**
   - 总结核心方法论（3-5 条）
   - 提出延伸思考或未来探索方向
   - 可加入互动引导

**禁止出现的内容**：
- ❌ "在当今时代""众所周知"等废话
- ❌ 纯理论堆砌，没有实操细节
- ❌ 流水账式的步骤罗列
- ❌ 结尾空喊口号"让我们一起"

**输出格式**：
```
【资深撰稿人初稿】
[完整文章内容，标注字数]
【创作说明】
- 使用的叙事手法：...
- 重点案例：...
- 创新点：...
```

---

### 第四阶段：技术审核

**编辑部 - 技术审核编辑** 发言：

逐段审查，输出修改意见：

```
【技术审核】
✓ 准确的技术点：[列出]
⚠ 需要修正的问题：
- 第 X 段：[问题描述] → 建议：[修改方案]
- ...
- 建议补充的内容：[具体说明]
修改后版本：[输出修改后的完整文章]
```

---

### 第五阶段：文字打磨

**编辑部 - 文字编辑** 发言：

专注语言打磨：

```
【文字编辑】
优化项：
1. 句式调整：[举例说明]
2. 用词精准化：[举例说明]
3. 节奏优化：[说明调整逻辑]
修改后版本：[输出修改后的完整文章]
```

---

### 第六阶段：用户体验优化

**编辑部 - 用户体验编辑** 发言：

从读者角度审视文章：

```
【用户体验编辑】
优化建议：
1. 信息层次：[如"第 X 节需要增加小标题"]
2. 视觉呈现：[如"代码块需要添加注释"]
3. 互动设计：[如"结尾可加入实践任务"]
修改后版本：[输出修改后的完整文章]
```

---

### 第七阶段：终审发布

**编辑部 - 终审官** 发言：

全面质检，做出发布决策：

```
【终审官】
质检结果：
✓ 优秀项：[列出]
⚠ 需要修正的问题：[列出]
发布决策：[通过/修改后发布/不通过]
决策理由：[详细说明]
最终修改建议：[如需要修改]
最终版本：[输出最终完整文章]
```

## 📤 输出标准

### 最终交付物

1. **完整文章**（终审官最终版本）
2. **创作过程记录**（7 个阶段的完整输出）
3. **质量报告**（终审官质检结果）

### 文章质量标准

| 维度 | 标准 |
|------|------|
| 开篇 | 200-300 字，场景引入，避免空话 |
| 主体 | 每章节 300-500 字，问题→方案→操作→效果 |
| 结尾 | 150-200 字，方法论总结 + 延伸思考 |
| 技术准确性 | 100% 准确，无技术错误 |
| 语言质量 | 流畅、精准、无废话 |
| 阅读体验 | 信息层次清晰、视觉呈现友好 |

## 📝 使用示例

### 示例 1：技术分享文章

**用户输入**：
```
帮我写篇文章，素材如下：
我最近用 Claude Code 的 memory 功能做了个记忆整合系统，每天凌晨 5 点自动执行，把短期记忆转化为长期记忆。用了 qwen3.5-plus 模型，效果不错。
```

**编辑部处理流程**：
1. 总编辑→确定目标受众（AI 开发者）、定位（经验分享）
2. 选题策划→提炼核心点（自动执行/记忆转化/模型选择）
3. 资深撰稿人→创作完整文章
4. 技术审核编辑→验证技术细节
5. 文字编辑→打磨语言
6. 用户体验编辑→优化阅读体验
7. 终审官→质检发布

---

### 示例 2：教程指南

**用户输入**：
```
把这个素材整理成教程：我配置 OpenClaw 飞书插件的步骤，包括 App ID/Secret 配置、OAuth 授权、权限设置
```

**编辑部处理**：
- 定位为"教程指南"
- 要求每个步骤包含：操作目的→具体操作→预期结果→常见问题
- 加入截图位置建议
- 结尾加入 FAQ 环节

## 🔧 技术实现

### 文件位置

```
agents/编辑部 - 总编辑/agent.md
agents/编辑部 - 选题策划/agent.md

agents/编辑部 - 用户体验编辑/agent.md
agents/编辑部 - 终审官/agent.md
```

### 调用方式

```javascript
// 1. 读取 7 个 Agent 提示词
const agents = await readAgents('编辑部');

// 2. 按顺序执行 7 阶段
const stage1 = await runAgent(agents.总编辑，素材);
const stage2 = await runAgent(agents.选题策划，stage1);

const stage4 = await runAgent(agents.技术审核编辑，stage3);
const stage5 = await runAgent(agents.文字编辑，stage4);
const stage6 = await runAgent(agents.用户体验编辑，stage5);
const stage7 = await runAgent(agents.终审官，stage6);

// 3. 输出最终文章
return stage7.最终版本;
```
