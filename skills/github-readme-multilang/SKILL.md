# GitHub README 多语言版本规范

**创建时间**：2026-04-07  
**来源**：用户王鲸明确要求  
**优先级**：最高（GitHub 发布基础要求）

---

## 🎯 核心原则

**每次发布 GitHub 项目时，多语言 README 是基础要求，必须遵守：**

1. ✅ **英语在上，中文在下** - 英语是国际通用语言，放在最前面
2. ✅ **单独文件** - 每个语言使用独立的 README 文件
3. ✅ **互相链接** - 每个 README 文件顶部有语言切换超链接
4. ✅ **内容完整** - 不能简化，保持内容完整性
5. ✅ **推荐组合** - 相关系统一起推荐安装

---

## 📁 文件命名规范

```
project/
├── README.md           # 英语版本（主文件，放在最上面）
├── README.zh.md        # 中文版本
├── README.ja.md        # 日文版本（可选）
├── README.es.md        # 西班牙文版本（可选）
└── ...
```

**语言代码规范**：
- `zh` = 中文
- `ja` = 日文
- `es` = 西班牙文
- `fr` = 法文
- `de` = 德文
- `ko` = 韩文
- `ru` = 俄文

## 📝 经验教训

**来源**：2026-04-07 Janus 发布教训

**问题**：
- 用户明确要求发布 GitHub 时多语言 README 是基础要求
- 但 Janus 项目只有中文版本 README.md
- 没有英语版本
- 没有语言切换链接
- 没有推荐 Dream 和 Lao 系统

**正确做法**：
1. 创建 README.md（英语主文件）
2. 创建 README.zh.md（中文文件）
3. 顶部添加语言切换链接
4. 英语内容完整，不简化
5. 添加 Dream 和 Lao 推荐章节
6. 互相链接

**教训**：
- 多语言 README 是 GitHub 项目的基础要求
- 英语必须放在最上面（国际通用）
- 每个语言单独文件，用超链接切换
- 相关系统一起推荐，发挥更强作用
- 不能为了省 token 简化内容
