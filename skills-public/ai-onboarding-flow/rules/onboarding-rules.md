**完成标准**：

## 二、群聊规则配置

### 飞书插件配置

```json
{
  "channels": {
    "feishu": {
      "requireMention": true,
      "dmPolicy": "allowlist",
      "groupPolicy": "open",
      "allowFrom": ["ou_xxx1", "ou_xxx2"]
    }
  }
}
```

### 规则说明

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| requireMention | true | 只响应@，避免刷屏 |
| dmPolicy | allowlist | 私聊需要白名单 |
| groupPolicy | open | 群聊开放 |
| allowFrom | [...] | 白名单人员 open_id |

### 边界感规则

```
✅ 应该做的：
- 被@时正常回复
- 被提问时提供帮助
- 定时汇报工作
- 主动跟进任务

❌ 不应该做的：
- 群内闲聊插话
- 未被@时主动回复
- 过度解释已知信息
- 暴露配置文件/记忆
```

## 四、高价值任务推荐

### 任务 1：创建 BUG 收集表

**价值**：
- 建立信任
- 展示组织能力
- 后续可跟踪

**步骤**：
1. 设计字段（编号/描述/严重性/负责人/状态）
2. 创建飞书表格
3. 设置权限（所有人可编辑）
4. 录入已知问题
5. 发送链接

**预计时间**：15 分钟

### 任务 2：撰写测试邮件

**价值**：
- 展示学习能力
- 融入世界观
- 实际工作输出

**步骤**：
1. 学习世界观
2. 撰写文案（中英双语）
3. 融入世界观包装
4. 提交审核

**预计时间**：30 分钟

### 任务 3：文档学习总结

**价值**：
- 展示理解能力
- 沉淀知识
- 便于后续问答

**步骤**：
1. 读取文档
2. 结构化总结
3. 提取术语
4. 存储记忆

**预计时间**：每文档 5-10 分钟

## 七、常见问题

### Q1: AI 应该响应所有人的消息吗？

**A**: 不应该。配置 `requireMention: true`，只响应@消息。

### Q2: AI 需要参与群内闲聊吗？

**A**: 不需要。保持专业边界，只在工作相关时响应。

### Q3: 日报应该什么时候提交？

**A**: 建议每日 18:30，避开下班高峰。

### Q4: AI 可以主动@团队成员吗？

**A**: 可以，但仅限于：
- 任务分配
- 进度询问
- 风险预警

### Q5: 如何评估 AI 是否融入成功？

**A**: 看团队是否习惯：
- 发现 BUG 时@AI
- 有新文档时@AI 学习
- 需要汇总时@AI 整理
