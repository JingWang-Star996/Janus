# 游戏配表规范与示例

## 配置表设计原则

### 1. 单一职责
每个配置表只负责一类数据，不要混用。

### 2. 唯一标识
每条记录必须有唯一 ID，建议格式：`类型_编号`

### 3. 类型明确
数值字段用数字，文本字段用文本，不要混用。

### 4. 引用规范
跨表引用使用 ID，不要使用名称。

### 5. 版本管理
配置表加版本号，便于追溯和回滚。

## 配表验证公式

### 基础验证（放在每行末尾）
```
=IF(AND(
  NOT(ISBLANK($A2)),           // ID 不为空
  NOT(ISBLANK($B2)),           // 名称不为空
  COUNTIF($A:$A, $A2)=1,       // ID 不重复
  $C2>=1, $C2<=5,              // 品质范围
  $D2>=0                       // 类型非负
), "✅", "⚠️ 请检查")
```

### 引用验证
```
// 验证技能 ID 是否存在
=IFERROR(
  IF(VLOOKUP($F2, config_skill!$A:$A, 1, FALSE)=$F2, "✅", "⚠️ 技能不存在"),
  "⚠️ 技能不存在"
)

// 验证多个技能 ID（用 | 分隔）
=IF($F2="", "✅",
  IFERROR(
    IF(AND(MAP(SPLIT($F2,"|"), LAMBDA(x, 
      NOT(ISERROR(VLOOKUP(x, config_skill!$A:$A, 1, FALSE)))
    ))), "✅", "⚠️ 技能 ID 错误"),
    "⚠️ 格式错误"
  )
)
```

### 数据完整性检查
```
// 检查所有宠物都有对应的技能
=IF(COUNTIF(config_skill!$A:$A, $F2)>0, "✅", "⚠️ 技能缺失")

// 检查进化链是否完整
=IF($I2="", "✅",
  IF(COUNTIF(config_pet!$A:$A, $I2)>0, "✅", "⚠️ 进化目标不存在")
)
```

## 配表工具推荐

### 1. 数据验证（飞书内置）
- 设置单元格验证规则
- 下拉选择枚举值
- 数值范围限制

### 2. 条件格式
- 重复值标红
- 空值高亮
- 异常值标记

### 3. 数据透视表
- 快速统计各类型数量
- 分析数值分布
- 发现异常数据

## 配表流程

1. **设计阶段**
   - 确定配置表结构
   - 定义字段类型和约束
   - 设计 ID 命名规范

2. **填写阶段**
   - 按模板填写数据
   - 实时验证数据
   - 及时保存备份

3. **验证阶段**
   - 运行验证公式
   - 检查引用完整性
   - 导出测试验证

4. **维护阶段**
   - 记录变更日志
   - 版本管理
   - 定期清理无用数据

## 常见错误与解决

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| #REF! | 引用无效 | 检查表格 URL 和范围 |
| #N/A | 查找不到 | 检查 ID 是否存在，数据类型是否一致 |
| #VALUE! | 类型错误 | 用 VALUE() 转换文本为数字 |
| #ERROR! | 公式语法错误 | 检查括号匹配和参数 |
| 循环引用 | 公式互相引用 | 检查公式逻辑，避免循环 |
