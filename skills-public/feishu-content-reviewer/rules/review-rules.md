## 一、脱敏级别定义

### 高风险（必须修改）🔴

**定义**：直接暴露商业机密、个人隐私、技术细节的内容

**处理方式**：必须修改，否则禁止发布

**示例**：
- 团队成员真实姓名（除作者外）
- 具体项目/系统名称
- 具体 BUG 代码/错误日志
- 服务器地址/内部环境名
- 商业合作细节

### 中风险（建议修改）🟡

**定义**：可能间接暴露项目信息的内容

**处理方式**：建议修改，需作者确认

**示例**：
- 团队规模具体数字
- 具体数值/数据指标
- 内部术语/代号
- 具体文档名称

### 低风险（可选优化）🟢

**定义**：一般性信息，泄露风险低

**处理方式**：可选优化，保留可读性

**示例**：
- 时间信息（具体时间点）
- 通用平台名称（飞书、OpenClaw）
- 工作流程描述
- 通用技术术语

## 三、审查流程

```
1. 接收文章
   ↓
2. 全文扫描（正则 + 关键词匹配）
   ↓
3. 标记可疑内容
   ↓
4. 风险分级（高/中/低）
   ↓
5. 生成审查报告
   ↓
6. 提供修改建议
   ↓
7. 作者确认/修改
   ↓
8. 创建飞书文档
   ↓
9. 输出链接 + 报告
```

## 四、例外情况

### 可保留的内容

| 类型 | 示例 | 说明 |
|------|------|------|
| 作者身份 | 游戏制作人王鲸 | 个人 IP，需要突出 |
| 通用工具 | OpenClaw、飞书 | 公开平台，可提及 |
| 工作流程 | BUG 跟踪、日报 | 通用管理方法 |
| 时间信息 | 2026-03-20、11:27 | 增强真实感 |
| 通用术语 | 前端、后端、策划 | 通用职位名 |

### 需要作者确认的内容

| 类型 | 说明 |
|------|------|
| 合作伙伴名称 | 可能需要签署 NDA |
| 具体数据指标 | 可能涉及商业机密 |
| 技术实现细节 | 可能暴露核心能力 |
| 客户/用户信息 | 隐私保护要求 |
