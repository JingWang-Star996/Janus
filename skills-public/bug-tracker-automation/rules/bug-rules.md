# BUG 管理规则

## 一、BUG 生命周期

```
发现 → 录入 → 分派 → 修复 → 验证 → 关闭
```

## 二、严重程度定义

| 级别 | 定义 | 响应时间 |
|------|------|----------|
| 致命 | 系统崩溃/数据丢失 | 立即 |
| 严重 | 核心功能不可用 | 4 小时 |
| 一般 | 功能异常但有 workaround | 24 小时 |
| 轻微 | UI/体验问题 | 72 小时 |

## 三、状态定义

| 状态 | 说明 |
|------|------|
| 待修复 | 等待负责人处理 |
| 修复中 | 负责人正在修复 |
| 已修复 | 修复完成，等待验证 |
| 已验证 | 测试验证通过 |
| 已关闭 | 流程结束 |

## 四、分派规则

- 前端问题 → 前端主程序
- 后端问题 → 后端主程序
- 策划问题 → 策划
- 美术问题 → 主美术
- 不确定 → 待分配
