## 📋 核心能力

### 1. BUG 表格创建

```
1. 自动设计字段
   - BUG 编号（自动生成）
   - 问题描述
   - 发现人
   - 发现时间
   - 严重程度
   - 状态
   - 负责人

2. 创建飞书表格
   - 设置列宽
   - 设置格式
   - 添加筛选视图

3. 权限配置
   - 所有人可编辑
   - 生成分享链接
```

### 2. BUG 录入

```
1. 从群消息提取
   - 识别 BUG 描述
   - 提取严重程度
   - 建议负责人

2. 自动录入表格
   - 生成编号
   - 填写字段
   - 标记状态

3. 通知相关人员
   - @发现人确认
   - @负责人跟进
```

### 3. BUG 跟踪

```
1. 状态更新提醒
   - 待修复 → 修复中 → 已修复 → 已验证

2. 负责人@提醒
   - 超期未处理
   - 需要协助
   - 等待验证

3. 进度统计报表
   - 按状态统计
   - 按负责人统计
   - 按严重程度统计
```

### 4. BUG 分析

```
1. 按模块统计
   - 前端
   - 后端
   - 策划
   - 美术

2. 按严重程度统计
   - 致命
   - 严重
   - 一般
   - 轻微

3. 趋势分析
   - 每日新增
   - 每日修复
   - 累积趋势
```

## ⚙️ 配置项

```yaml
# BUG 编号规则
bug_numbering:
  prefix: "BUG-"
  format: "001"  # 三位数字
  
# 严重程度定义
severity:
  - 致命：系统崩溃/数据丢失
  - 严重：核心功能不可用
  - 一般：功能异常但有 workaround
  - 轻微：UI/体验问题

# 状态流转
status_flow:
  - 待修复
  - 修复中
  - 已修复
  - 已验证
  - 已关闭

# 提醒规则
reminder:
  overdue_days: 2  # 超期 2 天提醒
  verify_remind: true  # 已修复提醒验证
```


### 2. 自动分派

```
用户：开启自动分派

AI:
✅ 自动分派已开启

**分派规则**：
- 前端问题 → 前端主程序
- 后端问题 → 后端主程序
- 策划问题 → 策划
- 美术问题 → 主美术
- 不确定 → 待分配

后续 BUG 会自动建议负责人。
```

### 3. 趋势预测

```
用户：预测一下 BUG 趋势

AI:
✅ 分析历史数据...

**趋势预测**：
- 本周预计新增：15-20 个
- 预计修复：10-15 个
- 累积趋势：下降

**风险预警**：
- 前端 BUG 增长率偏高
- 测试人力不足

**建议**：
- 加强前端代码审查
- 增加测试资源
```
