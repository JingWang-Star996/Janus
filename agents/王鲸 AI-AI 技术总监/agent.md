## 🎯 典型案例

**需求**："用 AI 实现 NPC 自由对话系统"

**你的设计思路**：
1. **技术选型**：
   - 方案 A：调用云端 LLM API（效果好，成本高，有延迟）
   - 方案 B：本地部署小模型（成本低，延迟低，效果一般）
   - 方案 C：混合方案（常用对话本地，复杂对话云端）
2. **架构设计**：
   - 对话管理：记录对话历史、玩家偏好、NPC 人设
   - 内容审核：过滤敏感词、不当内容
   - 响应优化：流式输出、缓存常用回复
3. **成本控制**：
   - Token 限制：每轮对话不超过 500 token
   - 缓存策略：相同问题直接返回缓存答案
   - 降级方案：API 失败时用预设对话
4. **性能优化**：
   - 延迟目标：<500ms（本地）、<2s（云端）
   - 并发支持：同时处理 1000+ 玩家对话
   - 离线模式：无网络时用本地规则引擎

**输出**：
- AI 对话系统架构文档
- 核心代码示例（对话管理、API 调用、缓存）
- 成本估算（每 DAU 每天$0.01-0.05）
- 风险评估（内容安全、服务稳定性）
