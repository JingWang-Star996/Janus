# Token 优化规范

**版本**：v1.0  
**创建日期**：2026-03-11  
**优先级**：⭐⭐⭐⭐⭐（最高）

---

## 📌 核心原则

**用户明确要求**：交互过程中注意减少不必要的 token 消耗

**所有 Agent 必须遵守**。

---

## 🎯 优化目标

| 目标 | 说明 |
|------|------|
| 💰 降低成本 | 减少 API 调用 token 消耗 |
| ⚡ 提高速度 | 减少处理时间 |
| 📦 节省窗口 | 留出更多空间给关键上下文 |
| 🎯 提升效率 | 更快完成任务 |

---

## ✅ 必须遵守的规范

### 1. 简洁回复

**原则**：能一句话说清楚的，不用两段

**示例**：

❌ **冗长**：
```
好的，我明白了！我现在就来帮你处理这个问题。
首先，我需要检查一下相关文件，看看具体情况。
让我先读取一下文件内容...
```

✅ **简洁**：
```
好的，我先检查相关文件。
```

---

### 2. 精准工具调用

**原则**：只调用必要的工具，能合并的不分开

**示例**：

❌ **浪费**：
```python
# 调用 3 次 read
read file.txt line 1-10
read file.txt line 11-20
read file.txt line 21-30
```

✅ **优化**：
```python
# 调用 1 次 read
read file.txt line 1-30
```

---

### 3. 合理分块

**原则**：大文件分块读取，用引用而非复制

**示例**：

❌ **浪费**：
```
这个文件内容是：
[复制全文 5000 字...]
```

✅ **优化**：
```
文件内容见 [FOLDER-RULES.md](file:///path/to/file.md)，共 6.4KB
关键内容：
- 创建 7 个分类目录
- 移动文件到对应目录
- 清理临时文件
```

---

### 4. 避免重复上下文

**原则**：不重复已知信息

**示例**：

❌ **浪费**：
```
根据我们之前的讨论，我们知道：
- 王鲸是游戏制作人 Agent
- 核心目标是次留 50%、7 留 20%
- 有三条路线：独立游戏、商业化小游戏、先就业再创业

现在我来回答你的问题...
```

✅ **优化**：
```
基于之前的讨论，直接说建议：

1. 根据你的情况（1 人、预算 10 万），推荐独立游戏路线
2. 具体步骤：...
```

---

### 5. 写文件 > Mental Notes

**原则**：重要信息写文件，不要"记在脑子里"

**示例**：

❌ **错误**：
```
我记住了这些信息，下次会用到。
[会话结束，信息丢失]
```

✅ **正确**：
```
已记录到 MEMORY.md：
- Token 优化原则
- 用户偏好
- 项目进度
```

---

### 6. 子代理优化

**原则**：指令简洁明确，只传递必要上下文

**示例**：

❌ **浪费**：
```python
sessions_spawn(task="""
你现在是一个专业的游戏制作人 AI 助手，名叫王鲸。
你有丰富的游戏开发经验，管理过 20+ 人的团队...
[500 字背景介绍...]

现在请回答用户的问题：...
""")
```

✅ **优化**：
```python
sessions_spawn(task="""
你是王鲸，资深游戏制作人。
核心能力：游戏设计咨询、项目管理、数据分析。
风格：数据驱动、直接幽默。

问题：[用户问题]
""")
```

## 📊 Token 消耗对比

### 场景 1：回复用户

| 方式 | Token 数 | 成本 |
|------|----------|------|
| 冗长回复 | 500 | 1x |
| 简洁回复 | 100 | 0.2x |

**节省**：80%

---

### 场景 2：文件读取

| 方式 | Token 数 | 成本 |
|------|----------|------|
| 复制全文 | 5000 | 1x |
| 引用 + 摘要 | 200 | 0.04x |

**节省**：96%

---

### 场景 3：工具调用

| 方式 | 调用次数 | 成本 |
|------|----------|------|
| 分开 3 次 | 3 | 1x |
| 合并 1 次 | 1 | 0.33x |

**节省**：67%

## 🛠️ 实用技巧

### 技巧 1：用表格代替文字

❌ **浪费**：
```
第一个目录是 projects，用来放项目文件。
第二个目录是 games，用来放游戏文件。
第三个目录是 outputs，用来放输出文件。
```

✅ **优化**：
```
| 目录 | 用途 |
|------|------|
| projects/ | 项目文件 |
| games/ | 游戏文件 |
| outputs/ | 输出文件 |
```

---

### 技巧 2：用列表代替段落

❌ **浪费**：
```
你需要做这些事情。首先，你需要检查文件。
然后，你需要修改代码。最后，你需要测试功能。
```

✅ **优化**：
```
步骤：
1. 检查文件
2. 修改代码
3. 测试功能
```

---

### 技巧 3：用链接代替复制

❌ **浪费**：
```
规范内容如下：
[复制 2000 字规范全文...]
```

✅ **优化**：
```
详见 [FOLDER-RULES.md](file:///path/to/file.md)
```

---

### 技巧 4：跳过显而易见的步骤

❌ **浪费**：
```
首先，我需要打开终端。
然后，我需要输入命令。
接着，我需要等待命令执行。
最后，我需要查看结果。
```

✅ **优化**：
```
执行命令，查看结果。
```

---

### 技巧 5：避免过度解释

❌ **浪费**：
```
我现在要使用 read 工具来读取文件。
read 工具可以读取文件内容，支持文本文件和图片。
对于文本文件，输出会被截断到 2000 行或 50KB。
让我来读取这个文件...
```

✅ **优化**：
```
读取文件。
```

## ⚠️ 常见错误

### 错误 1：过度礼貌

❌ "Great question! I'd be happy to help you with that!"
✅ 直接回答

---

### 错误 2：重复确认

❌ "你确认要删除这个文件吗？我再次确认一下..."
✅ "确认删除？[是/否]"

---

### 错误 3：详细解释工具

❌ "我现在使用 exec 工具，这个工具可以执行 shell 命令..."
✅ 直接执行

---

### 错误 4：复述用户问题

❌ "你的问题是如何整理工作区，让我来回答..."
✅ 直接回答

---

### 错误 5：过度总结

❌ "总的来说，我们讨论了 A、B、C 三个方面..."
✅ 只在必要时总结
