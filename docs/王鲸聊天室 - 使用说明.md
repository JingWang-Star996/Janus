## 工作原理

```
用户 (浏览器) → 服务器 → wangjing-messages.jsonl
                                    ↓
                              主会话查看
                                    ↓
                              王鲸回复
                                    ↓
用户 (浏览器) ← 轮询 ← wangjing-reply.txt
```

1. **服务器** 接收用户消息，写入 `wangjing-messages.jsonl`
2. **你（主会话）** 查看这个消息文件
3. **你用王鲸的身份回复**
4. **回复写入** `wangjing-reply.txt`
5. **浏览器轮询获取回复**

## 🎯 使用流程

1. 启动服务器：`python3 wangjing-server.py`
2. 浏览器访问：`http://localhost:8767`

1. 查看新消息：`cat wangjing-messages.jsonl`

## 常见问题

### Q: 为什么需要手动回复？
A: 这是最简单可靠的方案，不需要复杂的 API 集成，保证中文显示正常。

### Q: 消息发送后一直显示"正在思考"？
A: 检查服务器是否运行，然后在主会话查看消息并回复。

### Q: 可以自定义界面吗？
A: 可以！直接编辑 `wangjing-chat.html`。

### Q: 端口 8767 被占用怎么办？
A: 修改 `wangjing-server.py` 中的 `PORT = 8767`。
