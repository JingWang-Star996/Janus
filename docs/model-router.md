# Model Router - 模型路由配置

根据任务类型自动选择最合适的模型。

## 模型选择规则

| 任务类型 | 推荐模型 | 理由 |
|---------|---------|------|
| 通用对话/日常问题 | `qwen3.5-plus` | 平衡性能和速度 |
| 复杂编程任务 | `qwen3-coder-plus` | 专为代码优化 |
| 快速代码补全 | `qwen3-coder-next` | 响应快 |
| 深度推理/数学 | `qwen3-max-2026-01-23` | 最强推理能力 |
| 长文档分析 | `qwen3.5-plus` 或 `kimi-k2.5` | 大上下文窗口 |
| 视觉理解 | `qwen3.5-plus` 或 `kimi-k2.5` | 支持图像输入 |
| GLM 偏好任务 | `glm-5` | 智谱模型特长 |
| 轻量任务 | `glm-4.7` | 速度快 |
| 超长上下文 | `MiniMax-M2.5` | 131k 输出 tokens |

## 使用方式

**当前模式：自动切换** ✅

当收到任务时：
1. 判断任务类型
2. 使用 `sessions_spawn` 创建子 agent，指定对应模型
3. 子 agent 完成任务后返回结果

**用户偏好：** jing 已启用自动模型切换（2026-03-10）

## 示例

```
任务："帮我写一个 React 组件"
→ 使用 qwen3-coder-plus

任务："解这道数学题"
→ 使用 qwen3-max-2026-01-23

任务："总结一下这个 PDF"
→ 使用 kimi-k2.5
```
