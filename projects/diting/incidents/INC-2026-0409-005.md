## 5. 根因分析（5 Why）

### Q1: 为什么 npm 包被意外删除？
→ 多次尝试 `rm -rf /home/z3129119/.nvm/versions/node/v24.14.0/lib/node_modules/openclaw`

### Q2: 为什么要执行 rm -rf？
→ npm install 过程中出现 tarball 警告，命令被 SIGKILL 中断，AI 认为目录损坏需要清理重装

### Q3: 为什么 npm install 会失败？
→ 并发 exec 命令被 SIGKILL（可能是系统资源限制或超时），安装未完成就被中断

### Q4: 为什么会有并发 exec 被 SIGKILL？
→ AI 在同一个 turn 中启动了多个后台命令（npm update, npm install），它们竞争同一资源

### Q5: 为什么没有在升级前做好备份和风险评估？
→ AI 没有建立"升级系统组件"的标准操作流程

**根本原因**:
1. **没有升级 SOP** — 没有标准的 OpenClaw 升级流程
2. **并发命令冲突** — 多个 npm 命令同时执行导致目录损坏
3. **缺乏备份意识** — 升级前没有备份现有版本
4. **没有回滚计划** — 删除后无法快速恢复

## 6. OpenClaw 升级 SOP（新建）

```
收到升级请求
    ↓
1. 检查当前版本: openclaw --version
    ↓
2. 检查是否有运行中的 gateway: openclaw gateway status
    ↓
3. 备份当前配置: cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak
    ↓
4. 执行升级: npm install -g openclaw@latest (单一命令，不并发)
    ↓
5. 验证 CLI: openclaw --version
    ↓
6. 重启 Gateway: openclaw gateway restart
    ↓
7. 验证 Gateway: openclaw status
    ↓
8. 调用谛听检查升级过程
```
